## v0.4.0 — 2026-04-04

### CLI Provider Sessions

gl1tch now lets you open a direct chat session with any installed CLI provider — Claude Code, GitHub Copilot, OpenCode — without leaving the terminal. Create a session and start talking:

```text
/session new claude
```

Input goes straight to the provider. gl1tch stays out of the way.

![Claude Sonnet session inside gl1tch](/screenshots/console/claude_sonnet_chat.png)

When you need gl1tch's routing — to run a pipeline, check your brain, or switch sessions — prefix your message with `@glitch`:

```text
@glitch run my standup pipeline
@glitch what plugins do I have?
```

![Using @glitch routing inside a provider session](/screenshots/console/glitch_routing_in_session.png)

### Session Tab Bar

The main gl1tch session now shows as **gl1tch** in the footer tab bar. Provider sessions keep their name. You always know which context you're in.

![Session tab bar showing gl1tch and claude tabs](/screenshots/console/session_tabs.png)

Switch between sessions with `/s`:

```text
/s claude        # jump to claude session
/s               # list all sessions
```

### Notification Routing

Pipeline completions, clarification requests, and bot messages always route to the **gl1tch** session — they never interrupt your active provider chat. The gl1tch tab lights up when something needs your attention. Switch over when you're ready.

### Subprocess Job Runner

Pipelines and daemon supervisors now run as direct subprocesses. No tmux required for background work — gl1tch manages its own job lifecycle.

### /clear Resets Conversation

`/clear` now resets the provider resume ID along with the message history. Your next message starts a completely fresh conversation with the provider.
