## v0.5.0 — 2026-04-05

### Claude Provider Plugin

A dedicated Claude plugin joins the roster alongside Copilot and OpenCode. Open a Claude session, talk directly to the model, and use `@glitch` to re-engage routing whenever you need pipelines or brain access.

### /briefing

`/briefing` gives you a snapshot of your current context: active session, open tasks, and relevant brain notes — all in one shot.

### /shell

`/shell` drops into a shell pass-through session. Raw commands, no routing overhead.

### Persistent Widget Sessions

Widget processes now live in their own named session. They stay running, receive notifications, and survive session switches.

### Pipeline Results

```text
glitch pipeline results
```

Lists output from past pipeline runs. Pipe it, grep it, use it.

### Fixes

- First-run setup on a clean machine now completes without manual intervention.
- Daemon supervisor kills orphaned processes from a previous run before starting fresh.
- SIGHUP and SIGTERM are handled — daemons shut down gracefully on signal.
