orcai plugins let you extend the pipeline with any tool. Tier 1 plugins are native Go — fast, typed, compile-time safe. Tier 2 plugins are sidecar YAML wrappers — wrap any CLI binary without writing a single line of Go.

TIER 1 — NATIVE GO

Implement the Plugin interface in Go. Full type safety, zero overhead, access to internal orcai APIs. Best for core tools, formatters, and anything performance-critical. 


// sdk/plugin/plugin.go
type Plugin interface {
    Name()        string
    Description() string
    Execute(ctx context.Context,
            input Input) (Output, error)
}

Register with plugin.Register() in your plugin's init() function. orcai discovers all registered plugins at startup.

TIER 2 — SIDECAR YAML

Drop a YAML file in ~/.config/glitch/plugins/. orcai's CliAdapter wraps the binary automatically — no code required. Perfect for ripgrep, jq, ffmpeg, pandoc, your own shell scripts. 


name: ripgrep
description: Fast file search
command: rg
args: ["--json"]
input_schema: |
  {"type":"string",
   "description":"Search pattern"}
output_schema: |
  {"type":"array",
   "description":"Match results"}

orcai shells out to the binary, passes stdin, captures stdout/stderr, and exposes the result as a typed pipeline node.

// Pipeline Example //

Wire plugins together in a YAML pipeline

name: code-review-pipeline
description: Search, analyze, and summarize code issues

steps:
  - id: search
    plugin: ripgrep
    input: "TODO|FIXME|HACK"
    args: ["--json", "./src"]

  - id: format
    plugin: jq
    input: "{{ steps.search.output }}"
    args: ["-r", ".[].data.text"]

  - id: analyze
    plugin: claude
    prompt: |
      Review these code comments and identify the most critical issues:
      {{ steps.format.output }}
    model: claude-opus-4-5
    max_tokens: 1000

output: "{{ steps.analyze.output }}"
