## Draft: Improve Pr's

### Pr Guidelines
#### 1. Format

*   **Code formatting:** Consistent use of line lengths, indentation, and whitespace throughout the codebase.
*   **Commit messages:**
    *   Brief summaries of changes made.
    *   Use imperative mood (e.g., "Fix bug in App.tsx").
    *   Limit to 50 characters or less.

#### 2. Testing

*   **Unit testing:** Include tests for new features and functionality.
*   **Integration testing:**
    *   Test both happy paths and error cases.
    *   Ensure all components are properly integrated.
*   **End-to-end (E2E) testing:**
    *   Verify the entire application flows as expected.

#### 3. Collaboration

*   **Code reviews:** Regularly review each other's code for quality and adherence to guidelines.
*   **Issue tracking:** Use a version control system like Git to track changes and collaborate on projects.

#### 4. Documentation

*   **API documentation:**
    *   Clearly describe functions, variables, and classes.
    *   Include examples of usage.
*   **Project documentation:**
    *   Provide an overview of the project's goals and features.
    *   Outline installation and deployment instructions.
